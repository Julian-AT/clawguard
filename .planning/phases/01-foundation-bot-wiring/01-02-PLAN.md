       ```
       a. Extract PR metadata from message.raw: owner (raw.repository.owner.login), repo (raw.repository.name), prNumber (raw.prNumber)
          Type message.raw as: { repository: { owner: { login: string }; name: string }; prNumber: number }
          If GitHubRawMessage is importable from @chat-adapter/github, use that instead.
       b. Post acknowledgment: await thread.post("🔍 Starting security review for PR #${prNumber}...")
       c. Subscribe to thread: await thread.subscribe()
       d. try block:
          - Fetch PR details via octokit.pulls.get({ owner, repo, pull_number: prNumber }) to get head.ref and base.ref
          - Call reviewPullRequest({ owner, repo, prBranch: pr.head.ref, baseBranch: pr.base.ref })
          - Store results via storeAuditResult({ key: `${owner}/${repo}/pr/${prNumber}`, data: { result: reviewResult, timestamp: new Date().toISOString(), pr: { owner, repo, number: prNumber, title: pr.title }, status: "complete" } })
          - Post completion: await thread.post("✅ Security review complete for PR #${prNumber}. Results stored.")
       e. catch block (per D-07):
          - Post friendly error: await thread.post("Something went wrong during the security review. Please try again.")
          - Do NOT expose error.message or error.stack in the PR thread
       ```

       **`bot.onSubscribedMessage` handler:**
       - Post: `"I've already reviewed this PR. @mention me on a new PR to start another review."`

    2. **Create `app/api/webhooks/github/route.ts`** with these exact components:

       **Imports:**
       ```typescript
       import { after } from "next/server";
       import { bot } from "@/lib/bot";
       import { redis } from "@/lib/redis";
       ```

       **Route config:**
       - `export const maxDuration = 300;` — 5 minutes for sandbox operations

       **`export async function POST(request: Request)`:**
       ```
       a. Read delivery ID (safe — headers don't consume body):
          const deliveryId = request.headers.get("x-github-delivery");
       b. Idempotency check via Redis SETNX with 1-hour TTL:
          if (deliveryId) {
            const key = `webhook:delivery:${deliveryId}`;
            const isNew = await redis.set(key, "1", { nx: true, ex: 3600 });
            if (!isNew) return new Response("OK", { status: 200 });
          }
       c. Get Chat SDK handler (signature verification is internal to the adapter):
          const handler = bot.webhooks.github;
          if (!handler) return new Response("GitHub adapter not configured", { status: 404 });
       d. Delegate with after() for background processing:
          return handler(request, { waitUntil: (task) => after(() => task) });
       ```

       CRITICAL: Do NOT call `request.json()` or `request.text()` before passing to the handler — the body stream is one-shot and the SDK needs it for signature verification.
