    **lib/fix/apply.ts** — Fast-path fix application per D-05. Named export `applyStoredFix`:
    - Takes a Sandbox instance and a Finding
    - Reads original file from sandbox via `sandbox.readFileToBuffer({ path: finding.location.file })`
    - Replaces `finding.fix.before` with `finding.fix.after` using `String.replace()`
    - IMPORTANT: Normalize whitespace before comparison -- trim both `fix.before` and the content lines, normalize line endings with `.replace(/\r\n/g, "\n")`. If exact match fails, try a line-by-line fuzzy match (trim each line and compare). This addresses Pitfall 3 from research.
    - Verify the replace actually changed something (compare original vs fixed content). If no change occurred, return `{ valid: false, content: originalContent, errors: "fix.before pattern not found in file" }`
    - Write fixed file to sandbox via `sandbox.writeFiles([{ path: filePath, content: Buffer.from(fixedContent) }])`
    - Call `runValidation(sandbox)` from validate.ts
    - If validation fails, restore original file via `sandbox.writeFiles([{ path: filePath, content: Buffer.from(originalContent) }])`
    - Return `ApplyResult` with valid, content (the fixed content for commit), and errors

    **lib/fix/validate.ts** — Validation tool auto-detection per D-07, D-08, D-09. Named export `runValidation`:
    - Takes a Sandbox instance
    - Returns `ValidationResult` with `passed`, `errors`, `tools`
    - Detection order (run ALL detected, gate on ALL passing per D-09):
      1. Check `tsconfig.json` exists via `sandbox.runCommand("ls", ["tsconfig.json"])` -- if exitCode 0, run `sandbox.runCommand("npx", ["tsc", "--noEmit"])`
      2. Check for ESLint configs: iterate `["eslint.config.js", "eslint.config.mjs", "eslint.config.ts", ".eslintrc.js", ".eslintrc.json", ".eslintrc.yml"]`, first match runs `sandbox.runCommand("npx", ["eslint", "--no-warn-ignored", "."])`
      3. Check `biome.json` via `sandbox.runCommand("ls", ["biome.json"])` -- if exitCode 0, run `sandbox.runCommand("npx", ["biome", "check", "."])`
      4. Check `package.json` for test script: read via `sandbox.runCommand("cat", ["package.json"])`, parse JSON, check `scripts.test` exists and is not `echo \"Error: no test specified\" && exit 1`. If valid, run `sandbox.runCommand("npm", ["test"])`
    - For each tool that fails (exitCode !== 0), collect error from `stderr()` into errors array
    - `passed` is true only when errors array is empty
    - `tools` lists all detected tool names (even if they passed)
    - Use try/catch around each tool check so one tool failure doesn't prevent others from running

    All modules use named exports, no default exports. Import types from `@/lib/analysis/types` and `@/lib/fix/types`. Use `import type` for type-only imports.

    **lib/fix/commit.ts** — Commit validated fix to PR branch per FIX-03. Named export `commitFixToGitHub`:
    ```typescript
    import type { Octokit } from "@octokit/rest";
    import type { Finding } from "@/lib/analysis/types";

    export async function commitFixToGitHub(
      octokit: Octokit,
      params: {
        owner: string;
        repo: string;
        branch: string;
        filePath: string;
        content: string;
        finding: Finding;
      }
    ): Promise<string>
    ```
    Implementation:
    1. Call `octokit.repos.getContent({ owner, repo, path: filePath, ref: branch })` to get current file SHA
    2. Guard: if `Array.isArray(data)`, throw `Error("Path is a directory, not a file")`
    3. Call `octokit.repos.createOrUpdateFileContents({...})` with:
       - `message`: `fix(security): ${finding.type} (${finding.cweId})\n\nAuto-fix applied by ClawGuard for ${finding.severity} finding.\nLocation: ${filePath}:${finding.location.line}`
       - `content`: `Buffer.from(params.content).toString("base64")`
       - `sha`: `currentFile.sha`
       - `branch`: params.branch
       - `committer`: `{ name: "ClawGuard Bot", email: "clawguard[bot]@users.noreply.github.com" }`
    4. Return `commitResult.commit.sha` (the new commit SHA string)
    - Always fetch SHA immediately before commit (Pitfall 2 -- never cache SHAs)

    Implementation:
    - Import `ToolLoopAgent`, `Output`, `stepCountIs` from `"ai"`
    - Import `gateway` from `"@ai-sdk/gateway"`
    - Import `createBashTool` from `"bash-tool"`
    - Import `z` from `"zod"`
    - Define `FixOutputSchema = z.object({ fixedCode: z.string(), explanation: z.string() })`
    - Create bash tools: `const { tools } = await createBashTool({ sandbox })`
    - Create ToolLoopAgent with:
      - `model: gateway("anthropic/claude-sonnet-4.6")`
      - `tools`
      - `output: Output.object({ schema: FixOutputSchema })`
      - `stopWhen: stepCountIs(15)`
      - `instructions`: system prompt that includes the finding type, CWE ID, file path, line number, description, previous validation errors, and instructions to read the full file, fix the specific vulnerability without changing unrelated code, maintain code style, and return complete fixed file content
    - Call `agent.generate({ prompt: "Fix the ${finding.type} vulnerability in ${finding.location.file}" })`
    - Write the generated `fixedCode` to the file in sandbox via `sandbox.writeFiles`
    - Call `runValidation(sandbox)` from validate.ts
    - If validation passes, return `{ valid: true, content: result.object.fixedCode, errors: "" }`
    - If validation fails, restore original file, return `{ valid: false, content: "", errors: validation.errors }`
    - Wrap entire agent call in try/catch -- on error, return `{ valid: false, content: "", errors: error message }`
