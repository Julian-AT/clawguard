    BetterAuth server instance with GitHub OAuth social provider and stateless cookie sessions:
    ```typescript
    import { betterAuth } from "better-auth";
    import { nextCookies } from "better-auth/next-js";

    export const auth = betterAuth({
      socialProviders: {
        github: {
          clientId: process.env.GITHUB_CLIENT_ID!,
          clientSecret: process.env.GITHUB_CLIENT_SECRET!,
        },
      },
      session: {
        cookieCache: {
          enabled: true,
          maxAge: 60 * 60 * 24, // 24 hours
        },
      },
      plugins: [nextCookies()],
    });
    ```

    IMPORTANT: BetterAuth may require a database adapter even for OAuth. If the betterAuth() constructor requires a `database` field, use a no-op/memory adapter or check if there's a `noDatabase` option. The research indicates stateless cookie-based sessions should work without a database for this hackathon scope. If it errors, add a minimal adapter — Upstash Redis can serve as the database via the appropriate adapter package.

  <action>
    Extend `lib/redis.ts` with three new exported functions for dashboard data access. Add these AFTER the existing `getAuditResult` function. Do NOT modify existing functions.

    **1. Add `RepoSummary` interface** (used by dashboard overview page):
    ```typescript
    export interface RepoSummary {
      owner: string;
      name: string;
      lastAudit: {
        timestamp: string;
        score: number;
        grade: string;
        findingCount: number;
        prNumber: number;
        prTitle: string;
      } | null;
    }
    ```

    **2. Add `scanAuditKeys(prefix: string): Promise<string[]>`** (per D-17, Pitfall 3):
    - Use cursor-based SCAN with pattern `${prefix}/pr/*`
    - Initialize cursor as `"0"` (string)
    - Loop: call `redis.scan(cursor, { match: \`${prefix}/pr/*\`, count: 100 })`
    - Upstash Redis SCAN returns `[nextCursor: string, keys: string[]]`
    - Continue while `cursor !== "0"` (string comparison per Pitfall 3)
    - Return accumulated keys array

    **3. Add `getRepoAudits(owner: string, repo: string): Promise<AuditData[]>`**:
    - Call `scanAuditKeys(\`${owner}/${repo}\`)`
    - If no keys, return empty array
    - Fetch all audit data with `Promise.all(keys.map(k => getAuditResult(k)))`
    - Filter: keep only non-null results where `status === "complete"`
    - Sort by timestamp descending (most recent first): `new Date(b.timestamp).getTime() - new Date(a.timestamp).getTime()`
    - Return filtered and sorted array

    **4. Add `getDashboardRepos(repos: { owner: string; name: string }[]): Promise<RepoSummary[]>`**:
    - Takes a list of repos (from GitHub API — the caller provides this)
    - For each repo, call `getRepoAudits(repo.owner, repo.name)`
    - Build `RepoSummary` from the most recent audit (first in the sorted array):
      - If audits exist: extract score, grade, finding count, pr number/title, timestamp
      - If no audits: set `lastAudit` to null
    - Return array of RepoSummary objects, sorted: repos with audits first (by most recent timestamp), repos without audits last
    - Use `Promise.all` for parallel fetching across repos

    Import note: Add `import { countBySeverity } from "@/lib/analysis/scoring"` if needed for finding counts, OR just use `auditData.result.findings.length` directly for simplicity.
