# Phase 5 — UI Design Contract

> Visual and interaction contract for the dashboard pages. Generated by gsd-ui-researcher, verified by gsd-ui-checker.
>
> Phase 5 has three sub-domains: Chat follow-up, Config system, and Dashboard.
> Only the **Dashboard** sub-domain has web UI surfaces. Chat posts to GitHub PR threads (no web UI). Config is backend-only (YAML parsing).

## Spacing Scale

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding, badge padding |
| sm | 8px | Compact element spacing, card internal gaps |
| md | 16px | Default element spacing, card padding |
| lg | 24px | Section padding, card content spacing |
| xl | 32px | Layout gaps between major sections |
| 2xl | 48px | Page vertical padding |
| 3xl | 64px | Not used in this phase |

## Typography

## Color

All colors use shadcn CSS variable tokens (oklch values defined in `app/globals.css`).

| Role | Token | Dark Mode Value | Usage |
|------|-------|-----------------|-------|
| Dominant (60%) | `--background` | `oklch(0.145 0 0)` | Page background, full-bleed surfaces |
| Secondary (30%) | `--card` / `--muted` | `oklch(0.205 0 0)` / `oklch(0.269 0 0)` | Repo cards, table rows, nav bar, sidebar |
| Accent (10%) | `--primary` | `oklch(0.922 0 0)` | See reserved-for list below |
| Destructive | `--destructive` | `oklch(0.704 0.191 22.216)` | Sign-out action text (no confirmation needed), error states |
| Muted foreground | `--muted-foreground` | `oklch(0.708 0 0)` | Secondary text, timestamps, metadata |

Accent reserved for:
- "Sign in with GitHub" primary CTA button
- Score badge fill on repo cards (inherits Phase 4 score color mapping: green A/B, amber C, red D/F)
- Active nav indicator on dashboard
- Table row hover highlight
- "View Report" link text in audit table
- Active tab trigger underline

Severity badge colors (inherited from Phase 4 `severity-badges.tsx`):
- CRITICAL: `bg-red-500/15 text-red-400`
- HIGH: `bg-orange-500/15 text-orange-400`
- MEDIUM: `bg-yellow-500/15 text-yellow-400`
- LOW: `bg-blue-500/15 text-blue-400`
- INFO: `bg-zinc-500/15 text-zinc-400`

Score grade colors (inherited from Phase 4 `score-gauge.tsx`):
- A (90-100): green
- B (80-89): green
- C (70-79): amber/yellow
- D (60-69): red
- F (0-59): red

## Page Layout Contract

### Dashboard Overview (`/dashboard`)

```
+----------------------------------------------------------+
| Nav Bar                                    [Avatar] [v]   |
+----------------------------------------------------------+
| Page Heading: "Dashboard"                                 |
| Subtext: "{n} repositories audited"                       |
+----------------------------------------------------------+
| [Repo Card] [Repo Card] [Repo Card]                      |
| [Repo Card] [Repo Card] [Repo Card]                      |
|                                                           |
| -- or --                                                  |
|                                                           |
| [Empty State Card]                                        |
+----------------------------------------------------------+
```

- **Grid**: `grid grid-cols-1 sm:grid-cols-2 lg:grid-cols-3 gap-4`
- **Nav bar**: Full-width, `bg-card border-b border-border`, height 56px (`h-14`)
- **Nav content**: `max-w-6xl mx-auto px-6` (matches report layout)
- **Page content**: `max-w-6xl mx-auto px-6 py-8`

### Repo Card

| Element | Spec |
|---------|------|
| Container | shadcn `Card` with `hover:border-primary/30 transition-colors cursor-pointer` |
| Repo name | `text-sm font-semibold` truncated with `truncate` |
| Owner | `text-xs text-muted-foreground` above repo name |
| Score badge | shadcn `Badge` with grade-colored background (A/B green, C amber, D/F red) |
| Last audit | `text-xs text-muted-foreground` relative time (e.g., "2h ago") |
| Open findings | `text-xs text-muted-foreground` count with severity icon |
| Layout | Score badge top-right, repo info left, metadata bottom row |

### Per-Repo Detail (`/dashboard/[owner]/[repo]`)

```
+----------------------------------------------------------+
| Nav Bar                                    [Avatar] [v]   |
+----------------------------------------------------------+
| Breadcrumb: Dashboard > owner/repo                        |
| Page Heading: "owner/repo"                                |
| Subtext: "{n} audits"                                     |
+----------------------------------------------------------+
| Audit Table                                               |
| PR # | Title | Score | Severity | Date | Report Link     |
| ---- | ----- | ----- | -------- | ---- | -----------     |
| #42  | Add.. | A 95  | 0C 1H.. | 2h   | View Report ->  |
|                                                           |
| -- or --                                                  |
|                                                           |
| [Empty State]                                             |
+----------------------------------------------------------+
```

- **Table**: shadcn `Table` component. Dense rows, no zebra striping (zinc palette does not need it).
- **Table header**: `text-xs font-medium text-muted-foreground uppercase tracking-wide`
- **Table cell**: `text-sm` body text, `py-3 px-4` cell padding
- **PR column**: `font-mono text-xs` for PR number, `text-sm truncate max-w-xs` for title
- **Score column**: Grade letter + numeric score in a colored `Badge`
- **Severity column**: Inline severity count pills (reuse `SeverityBadges` pattern from Phase 4, compact variant)
- **Date column**: `text-xs text-muted-foreground` relative time
- **Report link**: `text-sm text-primary hover:underline` with arrow icon
- **Sort**: Most recent audit first (default, server-side)
- **Breadcrumb**: `text-sm text-muted-foreground` with `>` separator, current page not linked

### Auth / Sign-In

No dedicated sign-in page. Use a modal or redirect flow via BetterAuth.

- **Sign-in trigger**: When unauthenticated user visits `/dashboard/*`, redirect to BetterAuth GitHub OAuth flow
- **Sign-in button** (if shown on a landing/redirect page): shadcn `Button` default variant, full-width on mobile, with GitHub icon from lucide-react (`Github`), label "Sign in with GitHub"
- **Sign-out**: In avatar dropdown menu, `text-destructive` colored menu item labeled "Sign out"

### Nav Bar Component

| Element | Spec |
|---------|------|
| Container | `bg-card border-b border-border h-14 flex items-center` |
| Logo | `ShieldCheck` icon (lucide) + "ClawGuard" text, `text-sm font-semibold`, links to `/dashboard` |
| Right side | User avatar (`Avatar` component, 32px, GitHub profile image) + `DropdownMenu` |
| Dropdown items | "Dashboard" (link), separator, "Sign out" (`text-destructive`) |

## Interaction States

### Loading States

| Surface | Loading Pattern |
|---------|-----------------|
| Dashboard overview | 6x Skeleton cards in grid (same dimensions as RepoCard) |
| Per-repo detail | Skeleton table with 5 rows, matching column widths |
| Nav avatar | Skeleton circle 32px |

Use Next.js `loading.tsx` files for route-level loading states (matches Phase 4 pattern in `app/report/[owner]/[repo]/[pr]/loading.tsx`).

### Error States

| Surface | Error Pattern |
|---------|---------------|
| Dashboard overview | Centered Card with `ShieldAlert` icon + error copy (matches Phase 4 `error.tsx` pattern) |
| Per-repo detail | Same centered Card error pattern |
| Auth failure | Redirect to `/dashboard` with toast or inline error message |

### Empty States

| Surface | Pattern |
|---------|---------|
| Dashboard (no repos) | Centered Card with `ShieldCheck` icon, heading + body + CTA direction |
| Per-repo (no audits) | Inline message within the page content area |

### Hover / Focus

| Element | Hover | Focus |
|---------|-------|-------|
| Repo card | `border-primary/30` border transition | Standard focus ring via `outline-ring/50` |
| Table row | `bg-muted/50` background | Standard focus ring |
| Nav dropdown item | shadcn default DropdownMenu hover | shadcn default |
| View Report link | Underline appears | Standard focus ring |

## Accessibility Notes

- All interactive elements must have visible focus indicators (inherited from shadcn's `outline-ring/50` base style)
- Repo cards are wrapped in `<Link>` elements, not `onClick` divs
- Table uses semantic `<table>` markup (shadcn Table component provides this)
- Score badge colors include text labels (grade letter + number), not color-only communication
- Avatar includes `alt` text with GitHub username
- DropdownMenu items use proper `role="menuitem"` (provided by Radix)
