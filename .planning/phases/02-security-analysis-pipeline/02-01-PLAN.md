Create the foundational type system, scoring logic, and summary card builder for the security analysis pipeline.     **Write tests first (RED), then implement (GREEN).**

    **1. Create `lib/analysis/types.ts`:**

    Import `z` from `"zod"`. Define and export these schemas with `.describe()` on every field:

    `SeveritySchema`: `z.enum(["CRITICAL", "HIGH", "MEDIUM", "LOW", "INFO"])`
    `ConfidenceSchema`: `z.enum(["high", "medium", "low"])` (per D-08)

    `FindingSchema`: `z.object({...})` with these exact fields (per SCAN-05 + D-08):
    - `severity`: SeveritySchema
    - `type`: z.string() — vulnerability type like "sql-injection"
    - `location`: z.object({ file: z.string(), line: z.number() })
    - `cweId`: z.string() — e.g. "CWE-89"
    - `owaspCategory`: z.string() — e.g. "A03:2021-Injection"
    - `description`: z.string()
    - `attackScenario`: z.string()
    - `confidence`: ConfidenceSchema — per D-08
    - `dataFlow`: z.object({ source: z.string(), transform: z.string(), sink: z.string() })
    - `fix`: z.object({ before: z.string(), after: z.string() })
    - `complianceMapping`: z.object({ pciDss: z.string().optional(), soc2: z.string().optional(), hipaa: z.string().optional(), nist: z.string().optional(), owaspAsvs: z.string().optional() })

    `PhaseResultSchema`: `z.object({ summary: z.string(), findings: z.array(FindingSchema) })`

    `AuditResultSchema`: `z.object({
      phases: z.object({
        quality: PhaseResultSchema,
        vulnerability: PhaseResultSchema,
        threatModel: PhaseResultSchema,
      }),
      allFindings: z.array(FindingSchema),
      score: z.number(),
      grade: z.string(),
      severityCounts: z.record(z.string(), z.number()),
    })`

    Export inferred types: `type Finding = z.infer<typeof FindingSchema>`, same for Severity, Confidence, PhaseResult, AuditResult.

    IMPORTANT: Use `z.object()` only — NOT `z.interface()` (does not exist in zod@4.3.6, per research pitfall #2).

    **2. Create `lib/analysis/scoring.ts`:**

    Import `type { Finding }` from `"./types"`.

    Export `DEDUCTIONS` constant (per D-10): `{ CRITICAL: 25, HIGH: 15, MEDIUM: 8, LOW: 3, INFO: 1 } as const satisfies Record<string, number>`

    Export `GRADE_THRESHOLDS`: `[[90, "A"], [80, "B"], [70, "C"], [60, "D"]] as const`

    Export `calculateScore(findings: Finding[]): { score: number; grade: string }`:
    - `totalDeduction = findings.reduce((sum, f) => sum + (DEDUCTIONS[f.severity] ?? 0), 0)`
    - `score = Math.max(0, 100 - totalDeduction)`
    - `grade = GRADE_THRESHOLDS.find(([min]) => score >= min)?.[1] ?? "F"`
    - Return `{ score, grade }`

    Export `countBySeverity(findings: Finding[]): Record<string, number>`:
    - Initialize counts: `{ CRITICAL: 0, HIGH: 0, MEDIUM: 0, LOW: 0, INFO: 0 }`
    - Loop findings, increment `counts[f.severity]`
    - Return counts

    **3. Create test files** in `tests/analysis/` directory (create dir if needed).

    In test files, create a reusable `makeFinding(overrides)` helper that returns a complete valid Finding object with all required fields populated with realistic defaults. This avoids repeating 15+ fields in every test case.

    For scoring tests, create findings with specific severities using `makeFinding({ severity: "CRITICAL" })` etc.
